# Prompt｜Draft Reply Approval

## Purpose
把对请示事项的正式审批意见，起草为批复。

## Use when
- 用户要对请示进行正式回复
- 用户要表达同意 / 原则同意 / 不同意 / 附条件同意
- 用户要形成可留档的审批性回复文书

## Input to infer
- 原请示事项
- 来文单位 / 部门
- 批复结论
- 是否附条件
- 执行要求
- 生效或落实要求

## Writing rules
- 标题优先用“关于XXX的批复”
- 正文先点明收悉哪份请示，再给批复意见
- 批复结论必须明确，不能模糊漂浮
- 如为原则同意，应把前提条件和执行边界写清
- 语气正式、克制、可执行
- 不写成通知，也不写成决定

## Output contract
1. 批复成稿
2. 待补信息（如有）

## Self-check
- 是否明确回应了原请示
- 是否写清同意 / 不同意 / 条件
- 是否适合内部正式留档
