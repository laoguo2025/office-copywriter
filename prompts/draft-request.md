# Prompt｜Draft Request

## Purpose
把内部审批事项起草为正式请示。

## Use when
- 需要向上级或公司领导请求批准、授权、资源、预算、编制

## Input to infer
- 请示对象
- 请示事项
- 背景与原因
- 现状与必要性
- 金额/资源需求
- 希望批示内容

## Writing rules
- 标题优先使用“关于XXX的请示”
- 结构聚焦“背景—必要性—请示事项”
- 必须把要批什么写清楚
- 不要把请示写成单纯情况汇报
- 结尾使用“妥否，请批示”或等价表达
- 起草前优先判断主责部门口径，必要时参考：
  - `references/department-routing-rules.md`
  - `references/department-tone-baselines.md`
  - `references/department-vocabulary.md`
- 若请示偏行政保障、会务安排、日常事务资源申请，默认向行政口径收束
- 若请示偏人员编制、岗位调整、培训资源、用工事项，默认向人事口径收束
- 若请示偏设施维修、物资采购、后勤保障、安全整改资源，默认向后勤口径收束
- 若请示偏专项推进、跨部门协同资源、阶段任务支持，默认向综合运营口径收束
- 请示除了把“批什么”写清楚，也要让主责部门的理由结构和表达习惯自然成立，不要所有请示都写成一个口气

## Output contract
1. 请示成稿
2. 待补信息（如有）

## Self-check
- 是否明确请批事项
- 是否交代必要性
- 是否适合直接报领导审批
