# Prompt｜Draft Report

## Purpose
把工作进展、整改情况、阶段成果或异常情况汇总为报告。

## Use when
- 目标是汇报情况、反映问题、答复询问，而不是请批

## Input to infer
- 报告对象
- 报告主题
- 基本情况
- 进展/结果
- 问题与原因
- 下一步安排

## Writing rules
- 标题优先用“关于XXX的报告”
- 先给结论，再写事实、进展、问题、安排
- 数据或事实优先，少空话
- 不要把报告写成请示
- 结尾使用“特此报告”或等价表达
- 起草前优先判断主责部门口径，必要时参考：
  - `references/department-routing-rules.md`
  - `references/department-tone-baselines.md`
  - `references/department-vocabulary.md`
- 若报告偏事务组织、工作安排落实、报送反馈，默认向行政口径收束
- 若报告偏人员管理、培训落实、考勤纪律、组织执行，默认向人事口径收束
- 若报告偏安全检查、维修保障、设施运行、现场问题整改，默认向后勤口径收束
- 若报告偏跨部门推进、任务进展、节点复盘、闭环管理，默认向综合运营口径收束
- 同样是“报告”，不同部门应体现不同关注点和汇报重心，不要写成统一模板腔

## Output contract
1. 报告成稿
2. 待补信息（如有）

## Self-check
- 是否以汇报为核心而非请批
- 是否有事实、结果、问题、下一步
- 是否适合直接给领导看
