# Prompt｜Output Mode: Formal Issuance

## Purpose
把内部运营管理文稿收束为“正式发文版”，适用于内部下发、正式留档、统一口径执行。

## Use when
- 用户明确说要正式一点、发文、下发、红头口径、留档
- 文种属于通知 / 通报 / 决定 / 批复 / 制度 / 管理办法

## Output goals
- 标题规范
- 主送对象清晰
- 正文结构完整
- 执行要求明确
- 落款和日期口径完整

## Writing rules
- 标题优先采用“关于XXX的通知 / 通报 / 决定 / 批复”等标准格式
- 正文结构优先采用“事由 + 事项 + 要求”或“依据 + 决定/批复意见 + 执行要求”
- 语言正式、稳妥、简洁，不口语化
- 能明确责任、时限、执行边界的地方必须明确
- 不为了正式感堆砌空话套话

## Output contract
1. 正式发文版成稿
2. 待补信息（如有）

## Self-check
- 是否适合直接内部下发
- 是否具备留档感
- 是否避免口语和聊天腔
- 是否把执行要求写清楚
