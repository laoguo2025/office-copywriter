# Prompt｜Output Mode: Leadership Brief

## Purpose
把内部运营管理文稿收束为“领导汇报版”，适用于给领导快速审阅、把握重点、作出决策。

## Use when
- 用户明确说给领导看、用于汇报、压缩成重点版
- 文种属于请示 / 报告 / 情况说明 / 阶段汇报材料

## Output goals
- 先结论后展开
- 重点信息前置
- 风险、问题、建议、待决策事项清楚
- 避免冗长背景堆砌

## Writing rules
- 开头优先先写结论、现状判断或请批事项
- 中段按“事实 / 进展 / 问题 / 建议 / 需决策事项”组织
- 少写过程性流水账，多写提炼后的关键信息
- 如果是请示，必须把“请批什么”前置并说清
- 如果是报告，必须把结果、问题和下一步压缩到领导易扫读的结构里

## Output contract
1. 领导汇报版成稿
2. 待补信息（如有）

## Self-check
- 领导是否能在前几行看懂重点
- 是否保留了关键事实和决策点
- 是否压掉了低价值铺陈
