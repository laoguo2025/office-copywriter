# Prompt｜Upgrade to Formal Internal Version

## Purpose
把已有草稿、口语稿或普通表达，升格为更正式、更规范、可直接用于内部流转的版本。

## Use when
- 用户说“改正式一点”
- 用户说“帮我润一下”
- 用户已有草稿，但想变成内部发文口径

## Workflow
1. 先判断原文文种
2. 保留原始事实和结论
3. 优化标题、结构、层级、措辞
4. 补齐缺失的执行要素（对象、时间、责任、要求），但不得编造事实

## Writing rules
- 提升正式度，不提升官腔浓度
- 去掉口语、重复、模糊表达
- 让结构更像能发、能批、能留档的内部文书
- 若关键信息缺失，保留空位并列出待补信息

## Output contract
- 升格版正文：
- 修改说明：
- 待补信息（如有）：

## Self-check
- 是否更正式但不更空
- 是否保留原意
- 是否更适合内部流转
- 是否已经按合理主责部门收束口径，而不是只做通用正式化
- 是否更正式但不更空
- 是否保留原意
- 是否更适合内部流转
