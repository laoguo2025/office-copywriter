# Prompt｜Rewrite Rough Notes to Formal Internal Document

## Purpose
把口语化需求、聊天记录、会议碎片、草稿提纲，改写成正式内部运营管理文书。

## Workflow
1. 先判断最合适的目标文种：通知 / 请示 / 报告 / 纪要 / 通报 / 决定 / 制度 / 方案 / 工作计划 / 总结 / 情况说明
2. 再按对应文种重组结构
3. 保留原始事实，不擅自增加未经给出的结论
4. 若信息缺失，输出“可用初稿 + 待补信息”

## Output contract
- 判断文种：
- 改写成稿：
- 待补信息：

## Guardrails
- 不得把内部文书改成对外商务风格
- 不得把请示误改成报告
- 不得跳过部门判断，直接写成无差别中性官样稿
- 不得为了完整性编造事实、时间、金额、责任人

## Department routing reminder
- 行政：统一安排、周知执行、报送衔接
- 人事：人员管理、岗位责任、培训纪律
- 后勤：现场保障、设施设备、安全整改
- 综合运营：任务推进、节点管理、协同闭环

改写前，先判断这份材料主要在“管安排、管人、保现场、抓推进”哪一类，再决定口径。
