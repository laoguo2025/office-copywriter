# Prompt｜Draft Summary or Situation Statement

## Purpose
把既有工作复盘或已发生事项说明，起草为工作总结或情况说明。

## Use when
- 阶段工作总结
- 月度/年度工作总结
- 异常情况说明
- 管理例外说明

## Input to infer
- 文种更偏总结还是情况说明
- 基本情况或事件经过
- 主要成效/存在问题
- 原因分析
- 后续措施或下一步计划

## Writing rules
- 如果重点是复盘成效与问题，写成总结
- 如果重点是交代事实与原因，写成情况说明
- 总结要有成绩、问题、改进
- 情况说明要有事实、原因、措施
- 不得空泛拔高，不得回避关键问题
- 起草前优先判断主责部门口径，必要时参考：
  - `references/department-routing-rules.md`
  - `references/department-tone-baselines.md`
  - `references/department-vocabulary.md`
- 如果是情况说明：
  - 流程衔接、事务组织、报送异常类问题，默认向行政口径收束
  - 培训、考勤、人员管理、纪律执行类问题，默认向人事口径收束
  - 设施故障、现场保障、安全隐患、维修处置类问题，默认向后勤口径收束
  - 进度偏差、协同堵点、数据复盘、闭环缺失类问题，默认向综合运营口径收束
- 如果是总结，也应按主责部门调整关注点，不要把不同部门总结写成同一种复盘腔

## Output contract
- 文种判断：总结 / 情况说明
- 成稿：
- 待补信息（如有）：

## Self-check
- 是否选对文种
- 是否既讲事实又讲后续
- 是否适合直接提交或留档
