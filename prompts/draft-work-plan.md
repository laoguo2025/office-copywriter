# Prompt｜Draft Work Plan

## Purpose
把未来阶段性安排起草为工作计划，而不是实施方案或制度。

## Use when
- 年度工作计划
- 月度重点工作计划
- 部门阶段工作计划
- 节点推进计划

## Input to infer
- 计划周期
- 工作目标
- 重点任务
- 时间安排
- 责任分工
- 保障要求

## Writing rules
- 标题优先用“XXX工作计划”
- 强调时间顺序、重点任务、节点安排
- 不要写成长期制度
- 不要写成过重的“专项实施方案”腔调
- 适合管理层统筹或部门内部执行
- 起草前优先判断主责部门口径，必要时参考：
  - `references/department-routing-rules.md`
  - `references/department-tone-baselines.md`
  - `references/department-vocabulary.md`
- 行政口径的计划，应更强调服务安排、办公秩序、事务衔接、保障到位
- 人事口径的计划，应更强调人员管理、培训组织、制度执行、岗位责任
- 后勤口径的计划，应更强调巡检维修、设施设备、安全排查、现场保障
- 综合运营口径的计划，应更强调任务清单、责任分解、节点推进、过程闭环
- 同样的“月度重点工作计划”，应因部门不同而体现不同工作重心，不要写成一套空泛套话换标题

## Output contract
1. 工作计划成稿
2. 待补信息（如有）

## Self-check
- 是否体现计划周期
- 是否有任务、节点、责任
- 是否更像计划而不是方案或总结
