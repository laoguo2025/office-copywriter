# Prompt｜Review Internal Ops Document

## Purpose
审查一份企业内部运营管理文书，判断文种、结构、语气和执行性是否合格。

## Review dimensions
1. 文种是否正确
2. 结构是否完整
3. 语气是否匹配（上行/下行/平行）
4. 信息是否足够执行或审批
5. 是否存在空话、串型、模糊责任、缺少时间节点等问题

## Output contract
按以下格式输出：
- 文种判断：
- 主要问题：
- 修改建议：
- 修订版（可选，如用户要求直接改）

## Hard checks
- 请示不能没有请批事项
- 通知不能没有对象、时间、要求
- 纪要不能没有责任和时限
- 制度不能只有原则没有规则
- 情况说明不能只有解释没有措施
- 若内容明显属于行政 / 人事 / 后勤 / 综合运营中的某一类，不应审成无部门特征的中性模板稿

## Department routing reminder
- 行政：看是否把安排、衔接、报送要求写清
- 人事：看是否把人员管理、组织依据、纪律要求写清
- 后勤：看是否把现场保障、检查整改、设施风险写清
- 综合运营：看是否把任务拆解、节点推进、协同闭环写清
