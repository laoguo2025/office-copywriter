# doc-boundary-rules（文种边界规则）

## 1) 请示 vs 报告

### 请示
用于：
- 向上级请求批准、指示、授权

核心特征：
- 目标是“请批”
- 必须写清所请事项
- 结尾常见：`妥否，请批示` / `以上请示，恳请审批`

### 报告
用于：
- 向上级汇报工作、反映情况、答复询问

核心特征：
- 目标是“报情况”
- 重点是事实、进展、结果、问题
- 不以审批为核心目的
- 结尾常见：`特此报告`

### 快速判断
- 要批准 → 请示
- 要汇报 → 报告

---

## 2) 通知 vs 通报

### 通知
用于：
- 周知事项
- 布置工作
- 传达要求
- 发布安排

核心特征：
- 目标是让大家知道并执行

### 通报
用于：
- 表彰先进
- 批评错误
- 传达典型情况或重要精神

核心特征：
- 目标是“说明情况 + 形成警示/导向”

### 快速判断
- 重点是安排执行 → 通知
- 重点是情况传达/表扬批评 → 通报

---

## 3) 会议纪要 vs 会议记录

### 会议纪要
用于：
- 提炼会议议定事项
- 形成后续执行依据

核心特征：
- 不追求逐句还原
- 强调结论、共识、责任、时限

### 会议记录
用于：
- 记录发言过程与现场细节

核心特征：
- 更原始、更过程化
- 不一定形成正式执行口径

### 快速判断
- 要沉淀结论并执行 → 会议纪要
- 要保留过程原始记录 → 会议记录

---

## 4) 制度 / 办法 vs 方案

### 制度 / 管理办法
用于：
- 规范长期、重复性行为
- 形成稳定规则

核心特征：
- 长期适用
- 有条款、有适用范围、有约束性

### 方案
用于：
- 推动某一阶段、某一项目、某一专项工作

核心特征：
- 阶段性、任务性更强
- 强调目标、步骤、分工、保障

### 快速判断
- 长期规则 → 制度 / 办法
- 阶段推进 → 方案

---

## 5) 函 vs 通知

### 函
用于：
- 不相隶属单位间正式联系、商洽、询问、答复、邀请

核心特征：
- 对外或平行沟通
- 语气礼貌、边界稳妥

### 通知
用于：
- 内部下发、周知、执行要求

核心特征：
- 有组织层级或明确管理关系
- 更强调执行口径

### 快速判断
- 对外 / 平行沟通 → 函
- 对内周知执行 → 通知
