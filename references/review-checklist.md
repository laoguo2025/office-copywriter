# review-checklist（审稿检查清单）

## 一、先看文种是否正确
- 这是通知、请示、报告、纪要、通报、决定、制度、方案、总结，还是情况说明？
- 有没有把“请示”写成“报告”？
- 有没有把“会议纪要”写成“会议记录”？
- 有没有把阶段工作方案写成长期制度？

## 二、再看结构是否完整
- 标题是否明确
- 收文对象 / 适用对象是否清楚
- 事项背景是否交代到位
- 主要内容是否分层清楚
- 是否有执行要求 / 审批事项 / 议定事项 / 后续安排
- 是否有时间、生效点、责任分工

## 三、看语气是否合适
- 上行文是否谦稳、聚焦请批点
- 下行文是否明确、可执行、不拖泥带水
- 内部通报是否严肃但不过火
- 制度类是否客观、稳定、不口语化

## 四、看内容是否可执行
- 有没有明确“谁来做”
- 有没有明确“做什么”
- 有没有明确“何时完成”
- 有没有明确“怎么落实 / 怎么反馈”

## 五、看是否有常见问题
- 空话套话太多
- 逻辑顺序混乱
- 审批事项不明确
- 会议纪要没有责任与时限
- 通知没有对象、时间、要求
- 情况说明只有解释没有措施
- 制度只有原则没有规则与责任

## 六、输出前最后一眼
- 能不能直接发？
- 能不能直接交领导？
- 能不能直接存档留痕？

如果这三项里有一项答案是否定的，就还没改完。
