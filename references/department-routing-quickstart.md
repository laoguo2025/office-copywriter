# department-routing-quickstart（文种 × 输出模式 × 部门口径快速导航）

## 用途
这是 `office-copywriter` 当前最实用的总入口之一。

当你已经确定要写内部管理文书，但还需要快速决定：
- 用什么文种；
- 用什么输出模式；
- 偏哪个部门口径；

优先看这个文件，而不是在多份参考之间来回跳。

---

## 一、推荐默认决策顺序
1. 先判断文种；
2. 再判断输出模式；
3. 再判断主责部门；
4. 再调用对应 drafting prompt；
5. 最后用样例做快速校验。

简化成一句话就是：
**先判断“写什么”，再判断“怎么写”，最后判断“像谁写”。**

---

## 二、文种快速路由
- 通知：用于内部周知、统一执行
- 请示：用于向上请批、申请资源或授权
- 报告：用于汇报进展、反映情况、答复询问
- 通报：用于警示、表扬、检查反馈、结果导向
- 决定：用于正式部署、任免、奖惩、组织性决定
- 会议纪要：用于留痕会议结论、待办、责任和节点
- 制度 / 管理办法：用于长期规则、职责、流程规范
- 工作计划：用于未来阶段性安排和任务推进
- 总结 / 情况说明：用于复盘成效问题，或说明已发生事项

---

## 三、输出模式快速选型
- 快速可用稿：默认模式，适合直接起草内部可用版本
- 正式发文版：适合通知、通报、决定、正式制度文本
- 领导汇报版：适合请示、报告、需要上行汇报的稿件
- 制度条款版：适合制度、管理办法、规范细则
- 会议落地版：适合会议纪要，需要突出待办、责任人、节点

如果用户没明确说，就按“文种默认模式”走；
如果用户明确说“正式一点 / 给领导 / 做成制度条款 / 便于落地执行”，就切到对应模式。

---

## 四、部门口径快速判断
### 行政
适合：通知安排、会务组织、节假日安排、办公秩序、报送备案、日常事务统筹。
关键词：安排、通知、落实、报送、备案、值班、会务、秩序。

### 人事
适合：人员管理、培训、考勤、任免、绩效、岗位职责、组织纪律。
关键词：人员、岗位、培训、考勤、任命、纪律、绩效、交接。

### 后勤
适合：安全检查、设施设备、维修保障、食宿物业、现场管理、服务支撑。
关键词：巡查、维修、设施、设备、保障、整改、现场、隐患。

### 综合运营
适合：跨部门推进、重点任务统筹、节点管理、督办复盘、协同闭环。
关键词：统筹、推进、节点、闭环、协调、台账、督办、反馈。

---

## 五、最常用组合
- 通知 + 正式发文版 + 行政：节假日安排、值班安排、统一要求
- 通知 + 正式发文版 + 后勤：安全检查、现场秩序、保障要求
- 请示 + 领导汇报版 + 人事：编制、任免、培训资源、人岗安排
- 报告 + 领导汇报版 + 综合运营：专项推进、阶段成果、协同进展
- 通报 + 正式发文版 + 人事：考勤纪律、培训执行问题
- 会议纪要 + 会议落地版 + 综合运营：跨部门推进会、复盘会、协调会
- 制度 + 制度条款版 + 后勤：安全管理办法、设备巡检规范
- 工作计划 + 快速可用稿 + 综合运营：月度重点任务推进计划
- 情况说明 + 快速可用稿 + 行政 / 人事 / 后勤：按主责场景决定

## 七、当前使用原则
`office-copywriter` 现在不应再采用“先套一个通用模板，再临时修口气”的旧路径。

默认应采用：
**文种路由 → 输出模式 → 部门口径 → drafting prompt → 样例校验**

这样才能让真实出稿稳定体现差异，而不是每次靠碰运气。
