# doc-type-routing（内部运营管理文种路由）

## 1) 直接命中文种

用户明确说以下词时，直接按对应文种处理：
- 通知 → 通知
- 请示 / 申请审批 / 请求批准 → 请示
- 批复 / 审批回复 / 对请示回复 → 批复
- 报告 / 汇报 → 报告
- 纪要 / 会议纪要 → 会议纪要
- 通报 → 通报
- 决定 → 决定
- 制度 / 办法 / 规定 / 规范 → 制度 / 管理办法
- 方案 / 实施方案 / 工作方案 → 方案
- 计划 / 工作计划 / 年度计划 / 月度计划 → 工作计划
- 总结 / 复盘 → 总结
- 情况说明 / 异常说明 / 事实说明 → 情况说明

> 说明：本 skill 默认不以外部商务函、合作函、催办函为主路径；若用户强行指定外部函件需求，应提醒这已超出本 skill 的主职责范围。内部平行协同如确需正式留痕，可作为“内部协同函式沟通”补充路径处理。

## 2) 场景反推文种

### 通知
适用场景：
- 放假安排
- 会议通知
- 培训通知
- 入职安排
- 调岗通知
- 制度生效通知
- 工作要求下发
- 食堂 / 宿舍 / 后勤管理通知

### 请示
适用场景：
- 经费申请
- 预算申请
- 办公设备采购申请
- 招聘编制申请
- 维修 / 保障资源申请
- 事项审批申请

### 批复
适用场景：
- 对请示事项作正式回复
- 对采购、预算、编制、资源申请作审批回复
- 对下级部门报送事项作原则同意 / 不同意回复

### 报告
适用场景：
- 行政 / 人事 / 后勤工作进展汇报
- 阶段成果汇报
- 季度 / 月度工作报告
- 对上级答复说明

### 会议纪要
适用场景：
- 总经理办公会
- 项目推进会
- 部门协调会
- 形成议定事项并需执行的会议

### 通报
适用场景：
- 表彰先进
- 批评问题
- 考勤异常
- 安全检查情况通报
- 传达典型情况

### 决定
适用场景：
- 重要事项部署
- 组织架构调整
- 奖惩决定
- 任命决定

### 制度 / 管理办法
适用场景：
- 考勤管理
- 差旅报销
- 办公用品管理
- 员工行为规范
- 宿舍 / 车辆 / 资产管理

### 方案
适用场景：
- 培训实施方案
- 年度重点工作方案
- 专项整改方案
- 消防 / 防疫 / 安全专项安排
- 流程优化落地方案

### 工作计划
适用场景：
- 年度工作计划
- 月度重点工作计划
- 部门阶段工作计划
- 节点倒排计划
- 常规推进计划

### 总结
适用场景：
- 年终总结
- 专项工作总结
- 阶段复盘
- 月度 / 季度工作总结

### 情况说明
适用场景：
- 异常情况说明
- 原因说明与整改说明
- 人员异常说明
- 后勤故障说明

## 3) 冲突处理

- “申请 + 对上审批” 优先 `请示`
- “汇报 + 已完成/已发生情况” 优先 `报告`
- “对请示作正式回复” 优先 `批复`
- “通知大家执行” 优先 `通知`
- “会议结论要落地” 优先 `会议纪要`
- “表扬/批评/传达典型情况” 优先 `通报`
- “规范长期行为” 优先 `制度 / 办法`
- “阶段专项推进” 优先 `方案`
- “常规周期性安排” 优先 `工作计划`
- “复盘既有工作” 优先 `总结`
- “说明已发生事项与原因” 优先 `情况说明`

若仍冲突，只问一个问题：
> 这是要审批、要批复回复、要周知执行、要汇报情况、要形成周期性工作安排，还是要沉淀会议执行结论？
